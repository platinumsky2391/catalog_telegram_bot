# Инструкции для агента (Project Rules)

Ты — ведущий Fullstack-разработчик на Node.js и эксперт по Telegram Mini Apps. 
При написании и изменении кода для проекта строго следуй следующим архитектурным правилам:

1. **Окружение и Хостинг**: Проект развернут на виртуальном хостинге Beget под управлением модуля Phusion Passenger в режиме production (`NODE_ENV=production`).
2. **Структура директорий**: 
   - Исполняемые файлы бэкенда (`server.ts`, `server.cjs`), файл конфигурации пакетов (`package.json`) и переменные окружения (`.env`) находятся строго в корне папки домена: `/home/p/platines/bot-solien/`. Они изолированы от публичного доступа.
   - Папка `public_html` содержит исключительно файл `.htaccess` для маршрутизации Passenger. Никаких JS/TS файлов бэкенда внутри `public_html` быть не должно.
3. **Поиск путей**: В коде заложена логика поиска файла `.env` вверх по дереву каталогов относительно `process.cwd()`. При генерации путей к статическим файлам фронтенда (папка `dist`) используй конструкцию: `path.join(process.cwd(), "dist")`.
4. **Запуск бота**: Бот работает исключительно в режиме Long Polling (`bot.launch()`), перед запуском обязательно вызывается удаление вебхуков `bot.telegram.deleteWebhook({ drop_pending_updates: true })`. Не предлагай архитектуру на Webhooks.
5. **База данных**: Используется MySQL (`mysql2/promise`) с пулом соединений. Таблица `users` создается автоматически при старте сервера.
6. **Выходной формат бэкенда**: Главный файл компилируется в формат CommonJS с расширением `.cjs` (`server.cjs`).
